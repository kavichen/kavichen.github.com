Linux系統的在線求助man page與info page

#man page

  • NAME 簡短的命令.數據名稱說明
  • SYNOPSIS 簡短的命令執行語法(syntax)簡介
  • DESCRIPTION 較為完成的說明,這部份最好仔細看
  • OPTIONS 針對SYNOPSIS部份中,有列舉的所有可能的選項說明
  • COMMANDS 當這個程序或者軟件執行的時候,可以在此程序或軟件中中執行的命令
  • FILES 這個程序貨數據所是用或者參考或者連接到的某些文件
  • SEE ALSO 這個命令貨數據有相關的其他說明
  • EXAMPLE 一些可以參考的範例
  • BUGS 是否有相關的錯誤
  • 按下/date就可以超找文中的date關鍵字 #info page

第六章

Linux文件屬性

[-][rwx][r-x][r--]
 1  234  567  890
  • 1:代表這個文件名為目錄或者文件,本例中為文件(-)
  • 234:擁有者的權限
  • 567:同用戶組用戶權限
  • 890:其他用戶權限

如何改變文件屬性與權限

  1. chgrp
    • 改變文件所屬用戶組
  2. chown
    • 改變文件所有者
    • cp 源文件 目標文件之後,文件owner還是之前的,經常要在這個後面使用chown
    • chown [-R] 帳號名稱 文件或目錄 [-R]表示遞歸,即所有在改目錄下的文件都要改變owner.
  3. chmod
    • 改變文件權限
    • r:4 w:2 x:1 例子:rwx = 4+2+1 = 7
    • 例子:chmod 777 .bashrc

Linux文件種類與擴展名

文件種類:

  1. 普通文件 #純文本文件(ASCII) #二進制文件(binary) #數據格式文件(data) #目錄(directory)
    • 第一個屬性為[d] #連接文件(link)
    • 類似windows下的快捷方式,第一屬性為[l] #設備與設備文件
    • 塊(block)設備文件:就是一些存儲數據,以提供系統隨機訪問的接口設備,例如硬盤等.第一個屬性為[d]
    • 字符(character)設備文件:也即是一些串行端口的接口設備,例如鍵盤鼠標等.第一個屬性為[c]
    • 套接字(sockets):既然被成為數據接口,這種類型的文件通常被用在網絡的數據連接.第一屬性為[s]
    • 管道
      • FIFO是一個特殊文件類型,它主要目的在解決多個程序同時訪問一個文件所造成的錯誤問題.first in first out.第一屬性為[p]

Linux目錄配置標準:FHS

FHS需要有下面這些子目錄

  1. /bin 放置的是在單用戶維護模式下還能夠被操作的命令
  2. /boot 放置與開機有關的文件
  3. /dev 任何設備與接口設備都是以文件的形式存在于這個目錄當中
  4. /etc 系統主要的配置文件幾乎都在這個目錄內,例如人員的帳號密碼文件,各種服務器的啟始文件
  5. /home 這是系統默認的用戶文件夾
  6. /lib 系統的函數庫非常多,而/lib是放置在開機時會用到的函數庫
  7. /media 放置可刪除的設備,軟盤,光盤和DVD等
  8. /mnt 暫時需要掛載額外的設備,就要放置到這個文件夾下
  9. /opt 放置第三方軟件的目錄
  10. /root 系統管理員的主文件夾
  11. /sbin Linux有非常多的命令是用來設置系統環境的,這些命令只有root才能夠利用來"設置"系統,其他用戶最多只能用來"查詢".放在/sbin 下面的為開機過程中所需要的,裡面包括了開機,修復,還遠系統嗦需要的命令.
  12. /srv 可以視為「service」的縮寫,是一些網絡服務啟動之後,這些服務嗦需要取用的數據目錄
  13. /tmp 這是讓一般用戶或者正在執行的程序暫時放置文件的地方

絕對路徑 vs 相對路徑

  1. 絕對路徑:由根目錄(/)開始寫起的文件名或目錄名稱,例如/home/dmtsai/.bashrc
  2. 相對路徑:相對於目前路徑的文件名寫法.例如./home/dmtsai
    1. .代表當前的目錄
    2. ..代表上一層目錄

Linux文件與目錄管理

目錄的相關操作

  1. . 代表此層目錄
  2. ..代表上一層目錄
  3. -代表前一個工作目錄
  4. ~代表"目前用戶身份"所在的主目錄
  5. ~account代表account這個用戶的主文件夾(account是個帳戶名稱)

常用處理目錄的命令:

  1. cd: 切換目錄
  2. pwd: 顯示當前目錄
  3. mkdir: 新建一個新的目錄
    • mkdir [-p] 目錄名稱
      • 創建所有目錄
  4. rmdir: 刪除一個空的目錄
    • rmdir [-p] 目錄名稱
      • 參數p:連同上層「空的」目錄也一起刪除

關於執行文件路徑的變量:$PATH

  1. 先用root身份掠出查詢的路徑: #echo $PATH
  2. PATH這個變量的內容一定是由一對目目錄所組成的,每個目錄中間用(:)隔開
  3. 若要加入命令,就要修改PATH,例如# PATH="$PATH":/root,這樣就改到/root了

文件與目錄管理

  1. 查看文件與目錄:ls
  2. 複製,刪除與移動:cp,rm,mv
    • 注意-l和-s,硬鏈接與軟連接
  3. 取得路徑的文件名
    • #basename /etc/sysconfig/network
    • network <==文件名
  4. 取得目錄名稱
    • #dirname /etc/sysconfig/networ
    • /etc/sysconfig <== 取得目錄名

文件內容查閱

  1. cat:由第一行開始顯示文件內容
  2. tac:由最後一行開始顯示
  3. nl:顯示的時候,順便輸出行號
  4. less:與more相似,但是比more更好的是,它可以往前翻頁
  5. head:只看頭幾行
  6. tail:只看尾幾行
  7. od:以二進制的方式讀取文件內容.
  8. touch:修改文件時間或創建文件

文件默認權限:umask

umask是指定「目前用戶在新建文件或目錄時候的權限默認值」 

# umask
0022 <= 與一般權限有關的是後面三個數字
# umask -S
u=rwx,g=rx,o=rx

Hint - umask的分數指的是「該默認值要減掉的權限」,022指的是,user沒有減掉任何權限,而group和others的權限被拿掉了2.

  1. super block:記錄此文件系統的整體信息,包括inode/block的總量,使用量,剩餘量,以及文件系統的格式
